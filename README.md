備品貸出管理システム (Equipment Rental System)
1. プロジェクト概要
本プロジェクトは、効率的な備品管理と貸出プロセスの自動化を目的としたバックエンドシステムです。単なる機能実装に留まらず、データの整合性を保つためのバリデーションや、ユーザー体験（UX）を考慮した例外処理の共通化、そしてそれらを担保するテストコードの作成に重点を置いて開発しました。

2. 主要機能 (Key Features)
1) 会員管理および認証 (Auth & Member)
HttpSessionを活用したユーザー認証およびセッション管理の実装

入力値バリデーション: @Valid を活用し、パスワードの複雑性や必須入力項目のサーバーサイド検証を徹底

個人情報修正: ログイン状態に応じた会員情報の更新機能

✅ 備品貸出管理 (Rental Business Logic)
貸出制限ロジック: 過去の日付での予約禁止、および最大貸出期間（7日間）の制限を実装し、ビジネスルールの整合性を確保

ステータス管理: 備品の貸出状態（AVAILABLE/RENTED）をリアルタイムで反映し、二重貸出を防止

貸出履歴の高度化: ページング処理、検索、フィルタリング機能を実装し、大量のデータでもスムーズな閲覧が可能

✅ システムの安定化 (Stability)
例外処理の共通化: @ControllerAdvice を活用し、エラー発生時の適切なメッセージ表示とエラー画面への遷移制御を一本化

UI/UXの最適化: ログイン状態に応じたナビゲーションバーの動的表示およびログアウト時の機能制限を実装

3. 技術的課題と解決策 (Troubleshooting)
🔍 結合テストによるリダイレクト経路のバグ発見と修正
問題点: 会員情報修正の失敗時、入力フォームを維持せずホーム画面へリダイレクトされる設計ミスを特定しました。

解決策: MockMvc を用いた結合テスト（MemberUpdateIntegrationTest）を構築し、期待値（/members/edit）と実際値（/）の不一致を自動検出。リダイレクト先を明示的に修正することで、ユーザーが入力データを失わずに修正を継続できるUXを実現しました。

成果: 開発段階で論理的なバグを100%排除し、手動テストでは見落としがちなエッジケースを保護できました。

🧪 レイヤー別テスト戦略の導入
単体テスト: Mockito を活用し、外部依存（DB）を排除した状態で「過去日付予約禁止」などのコアロジックを高速に検証。

結合テスト: 実際のSpringコンテナ上でセッションとリダイレクト、DB反映までのライフサイクル全体を検証し、システムの信頼性を向上させました。

4. 技術スタック (Tech Stack)
Backend: Java 17, Spring Boot 3.x, Spring Data JPA

Database: MySQL (H2 for testing)

Testing: JUnit 5, Mockito, MockMvc, AssertJ

Frontend: Thymeleaf, Bootstrap 5

5. テスト結果 (Verification)
テスト成功率 100%: 全てのビジネスロジックおよびAPIエンドポイントに対するテストをパスしています。

自動ロールバック: @Transactional を適用し、テスト実行によるDBデータの汚染を完全に防止しています。
